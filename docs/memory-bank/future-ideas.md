## 🌟 v0.2.0 候选功能

### 1. 文件预览功能
- **需求**: 在浏览器中直接预览图片、PDF、视频
- **优先级**: 中
- **技术难点**: 需要前端解密后再渲染
- **评估**: 待 v0.1.0 稳定后考虑

### 2. 文件夹支持
- **需求**: 支持目录结构，而非扁平化文件列表
- **优先级**: 低
- **技术难点**: 需要重新设计数据模型
- **评估**: 与产品定位冲突（极简文件柜）

### 3. 批量下载（打包 ZIP）
- **需求**: 一次下载多个文件，自动打包成 ZIP
- **优先级**: 中
- **技术难点**: 需要实时压缩 + 流式传输
- **评估**: 有价值，v0.2.0 考虑

---

## 🔮 v0.3.0+ 远期想法

### 4. 移动端 App
- **需求**: iOS/Android 原生应用
- **优先级**: 低
- **技术难点**: 需要额外的开发资源
- **评估**: Web 端 PWA 可能更合适

### 5. WebDAV 协议支持
- **需求**: 支持 WebDAV，可作为网盘挂载
- **优先级**: 低
- **技术难点**: 需要实现完整的 WebDAV 协议
- **评估**: 偏离核心定位

### 6. 对象存储支持 (S3)
- **需求**: 支持 AWS S3、阿里云 OSS 等对象存储
- **优先级**: 高
- **技术难点**: 需要实现 storage.Engine 接口
- **评估**: 生产环境需要，v0.2.0 实现

### 7. 多租户支持
- **需求**: 支持多个独立的租户（如企业版）
- **优先级**: 低
- **技术难点**: 需要数据隔离、权限系统
- **评估**: 暂不考虑

## 🚫 明确不做的功能

### ❌ 社交功能
- 不做评论、点赞、关注
- 理由：偏离极简定位

### ❌ 在线编辑
- 不做文档在线编辑（如 Google Docs）
- 理由：技术复杂度高，非核心需求

### ❌ 版本控制
- 不做文件版本历史
- 理由：增加复杂度，用户需求不强
